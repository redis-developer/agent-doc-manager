# How to cache REST API responses Using Redis & NodeJS

![Ajeet Raina](https://cdn.builder.io/api/v1/image/assets%2Fbf70e6aa643f4e8db14c5b0c8dbba962%2F241cb4ece2bb4c0fb6df75c4495182cd)
![](/learn/_next/image?url=https%3A%2F%2Fcdn.builder.io%2Fapi%2Fv1%2Fimage%2Fassets%252Fbf70e6aa643f4e8db14c5b0c8dbba962%252F892aa19619da415da356ea3a61370284&w=1920&q=75)

This app returns the number of repositories a Github account has. When you first search for an account, the server calls Github's API to return the response. This can take some time. The server then adds the details of this slow response to Redis for future requests. When you search again, the next response comes directly from Redis cache instead of calling Github. The responses become much faster.

## Pre-requisite[#](/learn/howtos/caching#prerequisite)

### 1. Install NodeJS[#](/learn/howtos/caching#1-install-nodejs)

`brew install node`

### 2. Clone the repository[#](/learn/howtos/caching#2-clone-the-repository)

`git clone https://github.com/redis-developer/basic-caching-demo-nodejs`

### 3. Copy .env.sample to create .env[#](/learn/howtos/caching#3-copy-envsample-to-create-env)

`- REDIS_ENDPOINT_URI: Redis server URI

- REDIS_PASSWORD: Password to the server`

### 4. Run frontend[#](/learn/howtos/caching#4-run-frontend)

`cd client
yarn
yarn serve`

### 5. Run backend[#](/learn/howtos/caching#5-run-backend)

`yarn
yarn start`

Open up https://localhost:8081 and you can see a basic caching demo application up and running.
